Quantization Basics

Building a shared mental model before introducing AutoRound · ~15 min

What Is Quantization?

Quantization maps continuous floating-point values to a discrete set of integer values, reducing the number of bits needed to represent each weight or activation.

FP16 weight:  [0.234, -0.891, 1.203, -0.045, ...]   ← 16 bits
      ↓  quantize
INT4 weight:  [  3,    -7,     7,    -1,   ...]      ← 4 bits

Core Quantization Concepts

The Quantization Formula

scale = (xmax − xmin) / (2n − 1)
xquant = round((xfloat − zero_point) / scale)
xdequant = xquant × scale + zero_point

1. Scale (Δ) — Step size between quantized values. Larger → wider range but coarser steps.

2. Zero Point (Z) — Integer that maps to float zero. Symmetric (Z=0) for weights; asymmetric for activations.

3. Rounding — Maps float → nearest integer. Standard: round-to-nearest (RTN). AutoRound's insight: learned rounding — sometimes rounding up preserves more information.

4. Quantization Error — error = xfloat − xdequant. Measured as MSE, MAE, or task-specific accuracy drop. Not all errors are equal — some weights matter more.

AutoRound Algorithm: The Big Picture

Papers: arxiv.org/abs/2512.04746 · EMNLP 2024

Quantization is a discrete optimization problem — we can use gradient-based optimization if we make the rounding decision differentiable.

What AutoRound Optimizes

ParameterShapeControlsInitial
v (rounding)same as WRound up vs. down0 (nearest)

W_quant = clamp(round(W/scale + v), min_int, max_int)
W_dequant = W_quant * scale

Why It Works

  • Block-wise optimization — captures actual downstream impact
  • SignSGD — robust to noisy STE gradients
  • Joint optimization — rounding + clipping together

SignSGD: The Optimization Engine

Why SignSGD?

Standard SGD: w = w − lr × gradient
SignSGD:       w = w − lr × sign(gradient)
  • STE gradient magnitude is noisy — the sign is more reliable
  • Fixed step size naturally handles the discrete problem
  • Converges faster and more stably than Adam on this task

SignRound V1 → V2

V1 (EMNLP 2024): SignSGD on rounding only. Fixed quantization range.

V2 (arXiv 2025):

  • ① Adaptive mixed-precision — leverages gradient information and quantization-induced reconstruction errors to guide layer-wise bit allocation
  • ② Lightweight stabilization — loss filtering and pre-tuning scale search to improve tuning effectiveness in extremely low-bit regimes (2-bit, MXFP4)

SignRound V1 (EMNLP 2024)
aclanthology.org/2024.findings-emnlp.662

SignRound V2 (arXiv 2025)

AutoRound uses SignSGD because the sign of the gradient is more reliable than its magnitude when optimizing through a non-differentiable quantization operation.

QDQ Design: Why It Matters

QDQ / Fake Quantization (Tuning)

 Activation            Weight
    X_fp                W_fp
     │                   │
     ▼                   ▼
   Quant               Quant
     │                   │
     ▼                   ▼
   X_int4             W_int4
     │                   │
     ▼                   ▼
  DeQuant            DeQuant
     │                   │
     ▼                   ▼
  X_dq_fp ──→ FP GEMM ←── W_dq_fp

  Y = FP_GEMM(X_dq_fp, W_dq_fp)

DeQuant before GEMM → FP tensors → gradients flow through.

Real INT4 Kernel (Inference)

 Activation            Weight
    X_fp                W_fp
     │                   │
     ▼                   ▼
   Quant               Quant
     │                   │
     ▼                   ▼
 X_int4 scale_x     W_int4, scale_w
     │                   │
     └──→ INT4 GEMM ←────┘

 Y = INT4_GEMM(X_int4, W_int4, scale_x, scale_w)

Packed INT4 → fused dequant in kernel → no FP overhead.

VRAM Footprint: 4 GPUs → 1 GPU

BF16 FLUX transformer (23 GB) exceeds single B60 capacity → requires TP=4. W4A16 (7 GB) fits on 1 GPU with 19% headroom.

Memory headroom frees 2 GPUs → runs both CFG branches simultaneously → 1.55–1.67× speedup vs sequential BF16.

W4A16's value extends beyond memory savings — it enables parallel execution strategies that produce end-to-end speedups larger than raw dequantization overhead would predict.

Best Practices Overview

Choosing Quantization Schemes

Core principle: choose the scheme that has a mature kernel on your target platform. No kernel → no speedup, regardless of bit-width.

If you care about...Choose
⚠️ Check your device firstWhat data types does it natively support? What's the HBM bandwidth (decode bottleneck) and TFLOPS (prefill bottleneck)? Match the scheme to what your hardware actually accelerates.
Memory savingLower bits (W2A16/W3A16) → more KV cache headroom
Accuracy firstW4A16 g64, W8A8, or FP8 block-wise
PortabilityW4A16 g128 — most universal, kernel support everywhere
Long context / VLMsKV cache quantization (FP8/INT8) on top of weight quantization
